providers.json

Token Eye 核心配置文件。定义所有监控平台的 API 端点、解析规则、显示样式和自适应配色。

1. 顶层结构

{
  "providers": [ ... ],   // 平台列表（必填）
  "colors": { ... }       // 自适应配色（可选，有兜底）
}

2. Provider 配置

每个 provider 对象定义了一个监控平台。

2.1 通用字段

字段类型说明
id 必填string平台唯一标识，如 deepseek
name 必填string显示名称，如 DeepSeek
keychainService 必填stringmacOS Keychain 中存储 API Key 的服务名，如 DEEPSEEK_API_KEY

2.2 api — API 请求配置

字段类型默认说明
url 必填string—API 端点 URL
methodstringGETHTTP 方法
authHeaderstringAuthorization认证头名称
authPrefixstringBearer 认证前缀（注意末尾空格）

2.3 display — 显示配置

字段类型默认说明
unitstring—数值单位，如 ¥、%
labelstring—状态标签，如 余额、免费
nameColorhex自适应 default平台名称在菜单中的强调色

3. Parser 类型

三种 parser 对应不同 API 响应结构。

3.1 balance — 余额型

适用于返回余额信息的 API（如 DeepSeek）。

{
  "type": "balance",
  "fields": {
    "balance": "balance_infos.0.total_balance",  // 余额字段路径（点号分隔，数字=数组索引）
    "currency": "balance_infos.0.currency"            // 货币字段路径：CNY→¥，USD→$
  }
}
字段说明
balance用 . 分隔的 JSON 路径，定位余额值。数字表示数组索引
currency货币字段路径。默认 CNY

3.2 plan_usage — 用量型

适用于返回按模型分组的用量数据的 API（如 MiniMax）。

{
  "type": "plan_usage",
  "arrayPath": "model_remains",       // JSON 数组中存模型用量数据
  "fields": {
    "model": "model_name",                       // 模型标识字段
    "intervalPct": "current_interval_remaining_percent",  // 5小时窗口剩余百分比
    "intervalStatus": "current_interval_status",        // 状态：1=可用 2=耗尽临近 3=耗尽
    "weeklyPct": "current_weekly_remaining_percent",    // 周窗口剩余百分比
    "weeklyStatus": "current_weekly_status",          // 周窗口状态
    "intervalBoost": "interval_boost_permille",      // 限时加成（千分比，1000=无加成）
    "weeklyBoost": "weekly_boost_permille",          // 周加成（千分比）
    "resetMs": "remains_time"                       // 重置倒计时（毫秒）
  },
  "modelLabels": { ... },        // 模型 ID → 显示名映射
  "showModels": ["general","video"]    // 在菜单栏显示哪些模型
}

3.3 status — 状态型

适用于只需验证 Key 有效性的 API（如 MiMo）。

{
  "type": "status",
  "okField": "object",   // 响应中标记状态的字段路径
  "okValue": "list"      // 该字段匹配此值即表示 Key 有效
}
字段说明
okFieldJSON 路径，指向判断有效性的字段。留空则检查整个响应是否存在
okValue期望值。留空则只检查字段是否为非 null

4. 自适应配色

colors 段定义浅色/深色模式下的全局配色。脚本检测到外观后自动选用对应模式。内置兜底值，此段可选。

{
  "colors": {
    "dark": {
      "default":   "#ffffff",  // 默认文字（nameColor 兜底、菜单栏默认色）
      "secondary": "#aaaaaa",  // 次要文字（状态描述、子行信息）
      "muted":     "#888888",  // 弱化文字（时间戳、提示文字）
      "header":    "#FFD60A"   // Token Eye 标题色
    },
    "light": {
      "default":   "#1d1d1f",
      "secondary": "#6e6e73",
      "muted":     "#8e8e93",
      "header":    "#9a7600"
    }
  }
}
字段适用位置
defaultprovider nameColor 未设置时的兜底色、菜单项默认文字色
secondarystatus parser 的 "API Key 有效/无效" 文字、plan_usage 的子行文字
muted"上次更新" 时间戳、"未配置 Key" 提示行、providers.json 路径提示
header下拉菜单顶部 "Token Eye" 标题行

7. 完整示例

添加一个新平台的完整配置：

{
  "id": "my-platform",
  "name": "My Platform",
  "keychainService": "MY_API_KEY",
  "api": {
    "url": "https://api.example.com/v1/usage",
    "method": "GET"
  },
  "parser": {
    "type": "balance",
    "fields": {
      "balance": "data.credits",
      "currency": "data.currency"
    }
  },
  "display": {
    "unit": " credits",
    "label": "剩余",
    "nameColor": "#30D158"
  }
}

配置完成后将 API Key 存入 Keychain：

$ security add-generic-password -s MY_API_KEY -a "" -w "sk-your-key"

SwiftBar 下次刷新时自动加载，无需修改脚本。
